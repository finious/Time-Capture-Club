# Time-Capture-Club Demo Video Script

Date: 2026-07-18  
Status: Build Week demo script / under-three-minute target

## Demo Goal

Show one complete Time-Capture-Club release loop in under three minutes.

The viewer should understand:

- what problem the prototype solves
- how the loop works
- what the user receives
- how Codex helped build it
- why the receipt matters

## Opening Line

Most work tools show the final artifact. Time-Capture-Club preserves the invisible history that made the artifact trustworthy.

## 0:00-0:20 - Problem

Show the home screen.

Say:

People ship work, but the decisions, risks, tests, corrections, and next action disappear into chats, memory, screenshots, or somebody's head. Time-Capture-Club gives that invisible history a release loop.

Click: `Start a Release Loop`.

## 0:20-0:50 - Vision

Show the Vision panel.

Say:

First, the human names what the work is, who it serves, what it promises, and what counts as done enough. This keeps the dashboard from becoming the work. The human purpose stays in front.

Use sample data or fill one field.

## 0:50-1:15 - Structure

Show the Structure panel.

Say:

Next, we record where the work lives: source, project home, governing file, responsible human, and last known good state. This is the office function modern tools often erased: continuity.

## 1:15-1:45 - Change Review And Smoke Test

Show Change Review, then Smoke Test.

Say:

Then the release is inspected. What changed? What should not be touched? What actually passed? What failed? What was not verified? Time-Capture-Club does not fake confidence. Unknowns stay visible.

Set at least one smoke item to PASS and one to NOT VERIFIED.

## 1:45-2:10 - Release Gate

Show the Release Gate.

Say:

Automation can organize the evidence, but the human still makes the release decision: approved, blocked, or approved with known risks.

Select one release decision.

## 2:10-2:40 - Receipt

Show the Receipt panel.

Click: `Build Markdown Receipt`.

Say:

The output is a portable Markdown release receipt. It can be copied, downloaded, reviewed, taught, or used to return to the work later.

## 2:40-3:00 - Codex / Build Week Close

Show README or the app footer.

Say:

This static prototype was built during OpenAI Build Week with Codex as the coding partner. The human provided scope, judgment, and the release rules. Codex helped turn the governed loop into a working prototype.

Close with:

Professional work remembers how it became itself.

## Shot Checklist

- Home screen visible.
- Start a Release Loop clicked.
- Vision filled or sample loaded.
- Structure visible.
- Change Review visible.
- Smoke Test states visible.
- Release Gate selected.
- Markdown receipt generated.
- README or Codex mention visible.

## Do Not Show

- No long backstory.
- No full Working Room lore.
- No unrelated sites.
- No live Netlify repair.
- No feature roadmap ramble.

## Final Line

Build the loop. Film the loop. Receipt the loop. Submit the loop.
